Pipeline Overview

Step 1: CAD→SVG → Step 2: SVG→PNG → Step 3: VGGT → Step 4: Fitting → Step 5: Evaluation

Step 1: CAD to SVG
Load Program.json. Project 3D vertices to 2D using perspective projection (distance=30, elevation=30°, azimuth=45°, FOV=50°). Generate SVG with black strokes on white background.

Step 2: SVG to PNG
Convert SVG to 518×518 PNG using CairoSVG.

Step 3: VGGT Inference
Feed PNG to VGGT model. Output: depth map and camera pose. Apply scale factor = median (GT depths) / median (VGGT depths).

Step 4: 3D Stroke Fitting
Extract depths along strokes using Bresenham's algorithm. Convert camera-space depth to model-space z coordinate. Fit linear depth model along each stroke using Huber loss (threshold=1.0).

Step 5: Evaluation
Compare stroke endpoints. Calculate Mean Z error, Std Z error, Normalized error, Direction error.

Evaluation Metrics

Metric Description
Mean Z Error Average depth error for each stroke
For each stroke: error = (|z₁_pred - z₁_gt| + |z₂_pred - z₂_gt|) / 2
Mean Z error = average (error) for all strokes
Std Z Error Standard deviation of Z errors across all strokes
Std Z error = std (all stroke errors)
Normalized Error (%) Mean Z error normalized by scene depth range
Normalized error = Mean Z error / (max_z - min_z) × 100
Used for ranking to ensure fair comparison across different model sizes
Direction Error (°) Angle between predicted and ground truth stroke directions
θ = arccos (dot (normalize (pred_direction), normalize (gt_direction)))
Considers both forward and reverse directions
Depth Map MAE Mean Absolute Error of VGGT depth prediction at stroke pixels
MAE = mean (|depth_pred - depth_gt|) for all pixels covered by strokes
Pixel-level evaluation of raw VGGT output

Known Limitations

Endpoint Overlap in 2D Projection: Multiple 3D stroke endpoints may project to the same 2D pixel. Since VGGT predicts one depth value per pixel, endpoints at different depths cannot be distinguished. This primarily affects the 3D stroke reconstruction accuracy.
